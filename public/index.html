3) Phase-Based Operating Structure

Project lifecycle aligned to real field sequence:

  1. Earthwork
  2. Shell & Core
  3. Fitout
  4. Snag / De-snag
  5. Test & Commissioning
  6. Handover & Closeout

Each phase uses the same gate pattern:

Document Gate -> Field Gate -> Test Gate -> Compliance Gate -> Reporting Gate

Rule: No phase/activity release if critical gate criteria are not closed.

4) Submission List and Control Schema

Submission register must include mandatory fields for auditability and real-time traceability:

  • Submission ID, Phase, WBS Code, Area/Zone, Discipline
  • Document Type (Material / Method Statement / ITP / Test Certificate / Shop Drawing)
  • Spec Ref / Code Ref / Revision
  • Planned Submit Date / Required Approval Date
  • Status + Reviewer Route + Consolidated Comments
  • Linked Inspection Template + Linked Activity ID
  • Final Disposition + Effective Release Tag

Revision governance: latest approved revision controls field release; superseded revisions are locked from use.

5) Oracle Aconex-Style Workflow Logic

  1. Contractor submits package
  2. Auto-classification by phase/discipline/spec/WBS
  3. Parallel routing: QAQC + Discipline Lead + Consultant
  4. Comment consolidation and formal return
  5. Resubmission cycle (if required)
  6. Final disposition and release control
  7. Field checklist auto-trigger from approved docs
  8. Inspection/test execution + witness/hold handling
  9. NCR and corrective action loop
  10. KPI + red-flag + RCA escalation to management
