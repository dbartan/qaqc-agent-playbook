QAQC Agent System — Multi-Agent Construction Quality Control Engine

This system encodes 20+ years of real construction QAQC practice into a structured, AI-driven multi-agent framework. It is not a chatbot. It is a decision + validation engine for luxury hospitality construction projects (5-star hotel, branded residence, villa).

Version 1.0 L0/L1/L2 Agent Hierarchy Human-in-the-Loop Conflict Resolution 15 Master Rules

What This System Does

Submission Control

Contractor submission control (MAR, MS, ITP, Shop Drawing, test reports) with completeness gate + technical review.

Spec Compliance

Clause-by-clause compliance checking against project specifications, QMP, brand standards, and international standards.

Document Generation

Auto-generation of ITP, QCP, MS, NCR, Inspection Checklists — project-specific, site-ready, not generic.

Inspection Management

Pre/in-process/post/final inspections with hold points, witness points, and mandatory sign-off gates.

NCR Lifecycle

Non-conformance management: raise → classify severity → corrective action → verification → close.

Schedule Tracking

Submittal due dates, overdue flags, lead time calculation, and critical path impact assessment.

KPI & Reporting

Weekly QA reports, contractor scorecard, NCR trend analysis, risk heatmap by phase/trade.

Conflict Resolution

Agent disagreement detection → hold decision → escalation matrix → human QA Lead resolution.

15 Master Rules (Absolute)

These rules are absolute. No agent can override them. Defined in RULES.md.

R-01 No Assumption

Missing info = INCOMPLETE flag. Never fill gaps with assumptions.

R-02 Clause Reference

Every compliance decision must cite a specific spec clause or standard.

R-03 Sequence

No work without approved MAR + MS + ITP. Flag if missing.

R-04 Hold Point

Hold Points are absolute. Agent cannot clear — human sign-off mandatory.

R-05 Test Failure

Failed test = immediate NCR. Work suspended until NCR closed.

R-06 Conflict

Conflicting agents = HOLD all output + escalate to Human QA Lead.

R-07 Resubmission

Must address all previous comments. Without response = return without review.

R-08 Critical NCR

Immediate escalation to QAQC Manager. Stop work. Root cause + corrective action required.

R-09 Traceability

Every output carries: ID, phase, spec ref, drawing ref, date, status.

R-10 Draft

Agent docs = DRAFT. Not valid for execution until human approved.

R-11 Mockup (Luxury)

No mass production without workshop sample + site mockup + all sign-offs.

R-12 Material On Site

No install without approved MAR + MIR + storage condition check.

R-13 Overdue Submittal

7d: flag. 14d: escalate to PM. 21d: contractual notice trigger.

R-14 Standard Hierarchy

Stricter requirement always applies. Spec or standard — whichever is more stringent.

R-15 Learning Capture

Save best docs + lessons learned after phase handover for future reuse.

Approval Authority Matrix

Defined in policies/approval_matrix.md — governs which decisions agents can make vs. require human sign-off.

Authority LevelAgent Can Decide?Examples
AGENT — Automated Yes, no human needed Completeness check, draft generation, schedule alerts, register updates, conflict detection
HUMAN REVIEW — Agent recommends No, human must confirm Final submittal approval/rejection, NCR issuance, conditional approvals
MANDATORY SIGN-OFF — Human only Agent cannot override Concrete pour permit, structural release, hold point clearance, critical NCR closure, safety decisions

Mockup Approval Hierarchy (Luxury)

Level 1: Material Sample → Consultant approval
Level 2: Workshop Sample → Consultant + Client approval
Level 3: Site Mockup     → Brand rep + Consultant + Client review
Sign-off                 → Mass production release

RULE: No mass production without all three approvals.
